# PRD - AI功能调用错误复现与修复

## 背景
用户在 `D:\miaowu-os\deer-flow-main` 调用 AI 功能时仍报错，需要真实复现、归因并在可修复范围内完成修复与回归验证。

## 目标
1. 实际启动项目并完成一次完整 AI 功能调用链路，复现当前错误。
2. 输出可复核的复现步骤、报错信息、日志片段和时间点。
3. 将错误分为：
   - 项目代码缺陷（参数/逻辑/异常处理/契约漂移等）→ 实施修复
   - 上游服务异常（第三方 API / 远端服务故障）→ 只记录不修复
4. 修复后重复调用流程，给出修复前后对比和验证结论。

## 范围
- 目录：`D:\miaowu-os\deer-flow-main`
- 允许改动：与复现问题直接相关的 backend/frontend 代码、测试、必要文档
- 非目标：无关重构、跨模块清理、WSL 下前端依赖操作

## 约束
- local-dev 后端基址必须按项目约定使用 `http://127.0.0.1:8551`（前端 4560）。
- 对照原版 `D:\deer-flow-main` 同类逻辑，优先保持核心兼容。
- 若涉及小说链路，参考 `D:\miaowu-os\参考项目\MuMuAINovel-main` 的成熟实现。

## 验收标准
- 提供“复现步骤 + 日志证据 + 错误分类结论 + 修复说明 + 回归结果”。
- 若修复为代码问题：至少完成一次真实回归调用，证明错误消失或行为改善。
- 若仍存在上游故障：明确指出其归因与无法在仓库内修复的边界。
