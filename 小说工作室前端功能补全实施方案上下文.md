## 2. 技术栈差异对比

### 2.1 核心差异表

| 维度 | 参考项目（MuMuAINovel） | 主项目（Deer-Flow） | 迁移策略 |
|-----|----------------------|-------------------|---------|
| **UI框架** | Ant Design (antd) 5.x | shadcn/ui + Tailwind CSS | ⚠️ 需重写UI层 |
| **路由** | React Router v6 (`useNavigate`) | Next.js App Router (`useRouter`) | 替换路由Hook |
| **状态管理** | 直接调用API | @tanstack/react-query | 封装为Query Hooks |
| **HTTP客户端** | axios (`services/api.ts`) | 原生fetch (`novel-api.ts`) | 适配请求方法 |
| **样式方案** | 内联style对象 + antd token | Tailwind CSS类名 + cn()工具 | 转换为Tailwind |
| **图标库** | @ant-design/icons | lucide-react | 替换图标组件 |
| **TypeScript** | 宽松类型 | 严格类型（schemas.ts） | 对接类型系统 |

### 2.2 关键转换规则

#### 规则1: UI组件映射

```
参考项目 (Ant Design)          →  主项目 (shadcn/ui)
─────────────────────────────────────────────────

<Button>                       →  <Button variant="...">
<Input / <TextArea>            →  <Input /> / <Textarea />
<Select>                       →  <Select>
<Modal>                        →  <Dialog>
<Message> (antd全局)           →  toast() (sonner)
<Spin>                         →  <Spinner /> 或自定义
<Collapse>                     →  <Collapsible> (radix)
<Tag>                          →  <Badge variant="secondary">
<Empty>                        →  空状态自定义组件
<Progress>                     →  <Progress value={...} />
<Steps>                        →  自定义步骤指示器
<Upload.Dragger>               →  自定义拖拽上传
<Space>                        →  flex布局 (flex gap-...)
<Row/Col>                      →  Grid布局 (grid grid-cols-...)
```

### 4.1 职业体系 API ([careers.py](file:///d:/miaowu-os/deer-flow-main/backend/app/gateway/novel_migrated/api/careers.py))

**路由前缀**: `/api/careers`

| 方法 | 路径 | 说明 | 参数 | 返回值 |
|-----|------|------|------|-------|
| GET | `/api/careers?project_id=xxx` | 获取职业列表 | project_id | CareerListResponse { total, main_careers[], sub_careers[] } |
| POST | `/api/careers` | 创建职业 | CareerCreate body | CareerResponse |
| PUT | `/api/careers/{career_id}` | 更新职业 | career_id, CareerUpdate body | CareerResponse |
| DELETE | `/api/careers/{career_id}` | 删除职业 | career_id | { message } |
| GET | `/api/careers/{career_id}` | 获取职业详情 | career_id | CareerResponse |
| POST | `/api/careers/generate-system?project_id=xxx` | AI生成职业体系 (SSE流式) | project_id, main_career_count, sub_career_count, user_requirements, enable_mcp | SSE stream |
| GET | `/api/careers/character/{character_id}/careers` | 获取角色职业 | character_id | CharacterCareerResponse { main_career, sub_careers[] } |
| POST | `/api/careers/character/{character_id}/careers/main` | 设置主职业 | character_id, SetMainCareerRequest body | { message, career_name } |
| POST | `/api/careers/character/{character_id}/careers/sub` | 添加副职业 | character_id, AddSubCareerRequest body | { message, career_name } |
| PUT | `/api/careers/character/{character_id}/careers/{career_id}/stage` | 更新阶段 | character_id, career_id, UpdateCareerStageRequest body | { message, career_name, new_stage } |
| DELETE | `/api/careers/character/{character_id}/careers/{career_id}` | 删除副职业 | character_id, career_id | { message } |

**关键Schema字段**:
```python
class CareerCreate:
    project_id: str
    name: str
    type: Literal["main", "sub"]
    description: Optional[str]
    category: Optional[str]
    stages: List[CareerStage]  # [{level, name, description}]
    max_stage: int = 10 (main) / 5 (sub)
    requirements: Optional[str]
    special_abilities: Optional[str]
    worldview_rules: Optional[str]
    attribute_bonuses: Optional[Dict[str, str]]  # {"strength": "+10%"}
    source: Optional[str]  # "ai" or "manual"

class SetMainCareerRequest:
    career_id: str
    current_stage: int = 1
    started_at: datetime = now()
```
