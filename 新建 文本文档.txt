## 三、核心层文件 (src/core/novel/)
### 3.1 入口与导出

### 3.2 数据模式/类型定义
文件 用途 schemas.ts 核心数据模式定义文件 。使用 Zod 定义所有数据模型并导出 TypeScript 类型： Novel （小说）、 Volume （卷）、 Chapter （章节）、 Character （角色，含年龄/性别/外貌/性格/动机/背景/派系）、 Setting （场景，含类型/氛围/历史/特征）、 Faction （势力，含意识形态/目标/结构/资源）、 Item （物品，含类型/外貌/历史/能力）、 EntityRelationship （实体关系，5种类型）、 PromptTemplate （提示词模板，6种类型）、 TimelineEvent （时间线事件，3种类型）、 GraphLayout （图布局）、 ChatSession （聊天会话）、 OutlineNode （大纲节点）。还导出 CreateXxx 类型

### 3.3 数据库层
文件 用途 database.ts IndexedDB 数据库服务层 。基于 Dexie 构建 NovelDB 类，定义 12 张表（novels/volumes/chapters/characters/settings/factions/items/promptTemplates/relationships/timelineEvents/graphLayouts/snapshots/chatSessions）。 DatabaseService 类封装完整的 CRUD 操作：小说的保存/加载/更新；角色/势力/场景/物品/卷/章节的增删改；提示词模板管理（含激活状态切换）；关系管理；时间线事件管理；图布局保存；快照创建/查询；数据导出/导入/清空；聊天会话管理；仪表盘统计

### 3.4 查询层 (React Query)
文件 用途 queries.ts React Query 查询/变更钩子集合 。封装所有数据库操作为 React Query hooks，包括： useNovelQuery （加载单本小说）、 useAllNovelsQuery （加载所有小说列表）、 useDashboardStatsQuery （仪表盘统计）、 useUpdateNovelMutation / useUpdateChapterMutation （更新）、 useSaveSnapshotMutation / useChapterSnapshotsQuery （快照）、角色/势力/场景/物品/卷/章节的增删改 mutation、 usePromptTemplatesQuery / useActivePromptTemplateQuery （提示词模板查询）、 useRelationshipsQuery （关系查询）、 useTimelineEventsQuery （时间线事件查询）、 useGraphLayoutQuery / useSaveGraphLayoutMutation （图布局）、 useExportDataMutation / useImportDataMutation （数据导入导出）。所有 mutation 成功后自动 invalidate 相关查询

### 3.5 状态管理 (Zustand Stores)
文件 用途 useNovelStore.ts 小说 UI 状态管理 。Zustand + persist 中间件，持久化存储当前小说标题、活动章节ID、活动卷ID、视图模式（home/editor/graph/timeline/chat/outline/settings）。还管理脏内容标记（dirtyContent/isDirty）用于自动保存 useEditorStore.ts 编辑器 UI 状态管理 。Zustand store，管理工具栏开关、AI工具栏可见性、章节信息显示开关、字数/字符数计数 useAiPanelStore.ts AI 面板状态管理 。Zustand store，管理活动标签页（chat/generate/context）、AI 流式状态（isStreaming/latestChunk/seq/error）、选中文本、上下文实体列表。提供 startStreaming/addChunk/stopStreaming 等流式控制方法

### 3.6 AI 服务
文件 用途 ai-service.ts AI 服务核心类 。 NovelAiService 封装与后端 /api/novel/chat 的通信，支持流式/非流式响应、SSE 解析、中断控制。提供高级方法： generateOutline （生成大纲）、 continueWriting （续写）、 polishText （润色）、 expandScene （扩写场景）、 condenseText （精简）、 rewriteText （改写）、 extractEntities （实体提取，返回 JSON）

### 3.7 通用 Hooks
文件 用途 useDebounce.ts 防抖 Hook 。通用防抖函数，支持取消操作，用于编辑器内容自动保存（1秒防抖） useMediaQuery.ts 媒体查询 Hook 。响应式布局辅助，监听 CSS 媒体查询变化，用于判断桌面/平板/手机断点

### 3.8 工具函数 (utils/)
文件 用途 utils/index.ts 工具函数统一导出 。汇总导出所有子模块 utils/id.ts ID 生成工具 。提供带前缀的唯一 ID 生成： generateUniqueId （通用）、 generateCharacterId （char-前缀）、 generateChapterId （chapter-前缀）、 generateSettingId （setting-前缀）、 generateVersionId （version-前缀） utils/math.ts 数学/向量计算工具 。提供余弦相似度（ cosineSimilarity ）、欧几里得范数/距离（ euclideanNorm / euclideanDistance ）、向量归一化（ normalizeVector ），用于 embedding 向量计算 utils/token.ts Token 估算工具 。中文按1.2 token/字、英文按0.3 token/字估算，支持批量估算、使用率计算、按 token 限制截断（在句号处截断） utils/text-analysis.ts 文本分析工具 。从 HTML 提取 mention 的实体 ID（ extractMentionedIds ）、HTML 转纯文本摘要（ getPlainTextSnippet ）、提取章节细纲目标（ extractChapterGoal ，从 blockquote 标签提取）、清理小说 HTML 内容（ cleanNovelContent ） utils/jsonRepair.ts JSON 修复工具 。修复 AI 生成的畸形 JSON：去除 markdown 代码块标记/注释/尾逗号、提取有效 JSON 片段、补全未闭合括号/引号。 safeJsonParse 支持三级容错（直接解析 -> 修复后解析 -> 文本提取）。 validateJsonStructure 验证数据结构 utils/stream.ts SSE 流解析工具 。 parseSseStream 异步生成器解析 SSE 流； textToHtml 文本转 HTML； extractOpenAIContent / extractGeminiContent 分别提取 OpenAI/Gemini 格式的流式内容 utils/retry.ts 重试管理器 。 RetryManager 类实现指数退避重试策略（可配置最大次数/基础延迟/退避因子/最大延迟/抖动），智能判断哪些错误可重试（429限流最多2次，400/401/403/404/422不重试） utils/errorhandler.ts 错误处理器 。 ErrorHandler 类将各种错误标准化为 StandardError （8种错误类型：网络/API/验证/认证/授权/超时/中断/未知），支持日志记录、用户通知（toast）、可重试判断。导出全局实例 globalErrorHandler utils/image.ts 图片压缩工具 。 compressImage 将图片文件压缩为指定尺寸和质量的 JPEG base64，用于头像/封面等场景 utils/date.ts 日期格式化工具 。基于 date-fns 提供相对时间（ formatDateFromNow ，支持中英文）、日期字符串/日期时间字符串格式化、今天/昨天判断 utils/generationlogger.ts AI 生成日志记录器 。 GenerationLogger 类记录 AI 生成任务的完整生命周期（输入/原始输出/解析结果/token 用量/耗时/重试次数），最多保留50条日志。 analyzeError 智能分析错误类型（解析/网络/API限制/未知），生成用户友好提示。还提供统计功能（成功率等）

## 五、架构总结
整体架构采用 分层设计 ：

1. 数据层 ： schemas.ts （Zod 类型定义） -> database.ts （Dexie/IndexedDB 持久化） -> queries.ts （React Query 缓存与同步）
2. 状态层 ： useNovelStore （小说 UI 全局状态，持久化）、 useEditorStore （编辑器状态）、 useAiPanelStore （AI 面板与流式状态）
3. 服务层 ： ai-service.ts （AI API 通信，流式处理）、 utils/ （工具函数集）
4. 视图层 ： NovelWorkspace 作为主容器，按视图模式切换 Editor/Outline/Timeline/Graph/Settings 五大模块
5. 路由层 ：Next.js App Router，三个页面路由对应首页、工作区、独立编辑器
数据全部存储在浏览器端 IndexedDB 中，AI 功能通过后端 API 实现流式交互。

## 五、Hooks（自定义钩子）
文件 用途 useApiAction.ts 通用 API 操作 Hook，抽象异步操作的加载状态、错误处理、成功通知、请求取消、重试机制 useBrowserNavigation.ts 浏览器导航 Hook，拦截浏览器后退按钮，关闭模态框或返回编辑器视图 useCreationModal.tsx 通用创建模态框 Hook，简化各实体创建对话框的重复逻辑，提供 openCharacterModal/openSettingModal 等 useDebounce.ts 防抖 Hook，包含 useDebounce（值防抖）和 useDebouncedCallback（回调防抖，含 cancel 方法） useGlobalShortcuts.ts 全局快捷键 Hook，注册 Cmd+K（命令面板）、Cmd+S（保存）、Cmd+Shift+C/S/E（快速创建实体） useGraphData.ts 关系图数据适配器 Hook，将 IndexedDB 数据转换为 force-graph 可用的 nodes/links 格式，支持节点聚焦 useGraphLayout.ts 图布局持久化 Hook，管理节点位置的保存/加载/锁定/重置 useHybridSearch.ts 混合搜索 Hook，结合关键词匹配（实时）和向量语义检索（防抖），提供统一搜索结果 useMediaQuery.ts 响应式媒体查询 Hook，检测 CSS 媒体查询是否匹配（用于移动端适配） useMentionItems.ts 角色提及数据 Hook，获取格式化的角色列表用于编辑器 Mention useMentionOptions.ts 统一提及选项 Hook，聚合角色/场景/势力/物品四种实体类型的提及数据 useMutationForm.ts 通用表单 Mutation Hook，封装表单创建/提交/验证/React Query Mutation 绑定逻辑，含角色/章节/卷/势力/场景专用子 Hook useTheme.ts 主题管理 Hook，提供主题切换（light/dark/system 循环）和 localStorage 持久化 useUnsavedChangesWarning.ts 未保存变更警告 Hook，监听 beforeUnload 事件，防止用户误关闭导致数据丢失
