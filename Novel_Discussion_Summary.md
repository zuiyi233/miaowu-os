## 一、合规性检查结论（第一轮讨论）

### 1.1 小说创作流程检查

**结论：✅ 符合要求**

- 用户发起"帮我写一本小说"后，系统通过 `IntentRecognitionMiddleware` 创建内存会话，**不立即落库**
- 通过 `_FIELD_ORDER` 追问机制收集缺失字段（书名、类型、主题、受众、目标字数）
- 所有字段补齐后进入确认流程，用户确认后才执行 `_finalize_creation` 落库

## 三、小说创作流程全面评估（第三轮讨论）

### 3.1 核心差距（与行业最佳实践对比）

| 差距维度 | 当前状态 | 行业最佳实践 |
|-----------|---------|------------|
| 质量控制自动化 | 审核需手动触发 | 生成后自动审核，低分自动修复 |
| 长程一致性保障 | 仅单章分析 | 角色/物品/时间线全维度一致性引擎 |
| 上下文记忆管理 | 仅注入前1章摘要 | 分层摘要 + 状态快照 + 关键事件索引 |

### 3.2 五大流程断点

```
断点1: 选题构思 → 大纲设计
   问题：选题仅收集基础字段，无故事梗概/核心冲突输出

断点2: 大纲设计 → 章节撰写
   问题：大纲更新后，已生成章节不会自动标记一致性风险

断点3: 章节撰写 → 内容审核
   问题：审核需手动触发，用户经常跳过

断点4: 内容审核 → 修订润色
   问题：分析结果未自动转化为修订指令

断点5: 修订润色 → 最终定稿
   问题：无定稿检查清单，无版本锁定
```

### 3.3 最优先改进建议

| 优先级 | 建议 | 预期效果 |
|--------|------|---------|
| **P0** | 建立自动生成-审核-修订流水线 | 审核覆盖率从30%→100% |
| **P0** | 引入跨章一致性检查引擎 | 角色崩塌发现率提升80%+ |
| **P0** | 引入分层大纲体系 | 中期剧情跑偏率显著降低 |

## 四、多智能体架构分析（第四轮讨论）

### 4.1 主项目多智能体能力

**deerflow 具备完整的多智能体架构：**

| 组件 | 功能 |
|------|------|
| Lead Agent | 主智能体，接收用户请求并决策 |
| Subagent（子智能体） | 通过 `task_tool` 委派任务 |
| SubagentExecutor | 子智能体执行引擎 |
| SubagentConfig | 支持独立指定模型（`model: inherit` 或自定义） |
| SubagentLimitMiddleware | 限制并行子智能体数量（默认最多3个） |

### 4.2 "高低搭配"能力

```python
# SummarizationMiddleware 使用轻量模型
model = create_chat_model(thinking_enabled=False)  # 轻量模型做摘要

# SubagentConfig 支持独立模型配置
@dataclass
class SubagentConfig:
    model: str = "inherit"  # 可指定不同模型
```

### 4.3 小说模块现状

**结论：小说模块完全没有使用多智能体架构**

- 小说模块通过 `AIService` **直接调用模型**，无Agent决策层
- 所有功能（生成、审核、修订）使用**单一模型**
- 批量生成是简单的 `for` 循环，而非并行子智能体任务

### 4.4 优化建议

| 建议 | 方案 | 预期效果 |
|------|------|---------|
| 建立小说专用子智能体 | 在 deerflow Subagent 框架下定义 novel-writer / novel-critic | 任务隔离、质量提升 |
| 生成+审核双模型搭配 | 生成用用户模型，审核用强推理模型（DeepSeek-R1） | 审核质量显著提升 |
| 批量生成并行化 | 利用 `task_tool` 并行派发章节生成 | 耗时降低60-70% |

## 五、设置功能优化需求（当前任务）

### 5.1 现有设置架构

**主项目设置（前端）：**
- 位置：`frontend/src/components/workspace/settings/`
- 页面：AI供应商、工具、技能、记忆、外观、通知、关于
- 状态管理：`ai-provider-store.ts`（Zustand）

**小说模块设置（后端）：**
- 位置：`backend/app/gateway/novel_migrated/api/settings.py`
- 功能：AI模型配置、预设管理、SMTP配置、封面配置
- 状态管理：`Settings` 模型（数据库）

### 5.2 当前问题

1. **小说设置与主项目设置分离**：用户需在两个界面配置AI参数
2. **无模型组合搭配功能**：无法配置"生成模型+审核模型"的高低搭配
3. **无智能体参数配置**：temperature、max_tokens等仅支持全局设置，不支持按智能体/场景配置
4. **供应商选择不统一**：主项目前端有供应商管理，小说后端有独立的provider配置

### 5.3 优化目标

1. **增强小说智能体模型配置**：在设置页面实现模型组合搭配交互界面
2. **完善主项目设置页面**：添加供应商选择、模型选择、智能体参数调整
3. **整合设置功能**：将小说设置页独特功能迁移至主项目设置页，消除功能重叠
