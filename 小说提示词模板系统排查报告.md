**核心类**：`AIService`  
**文件**：`D:\miaowu-os\deer-flow-main\backend\app\gateway\novel_migrated\services\ai_service.py`

**桥接关系**：

### 1.3 调用关系矩阵

| 调用方向 | 状态 | 路径 |
|---------|------|------|
| 主项目 → 小说AI服务 | 已打通 | `ai_provider.py` → `AIService` |

| 小说模块 → 主项目AI配置 | 已打通 | `AIService` → `deerflow.config` |
| 小说模块 → 主项目数据库 | 隔离 | 使用独立 `novel_migrated.db`（设计如此） |

### 7.2 get_template 改造

```python
@classmethod
async def get_template(cls, template_key: str, user_id: str, db):
    if user_id and db:
        result = await db.execute(
            select(PromptTemplate).where(
                PromptTemplate.user_id == user_id,
                PromptTemplate.template_key == template_key,
                PromptTemplate.is_active == True
            )
        )
        custom = result.scalar_one_or_none()
        if custom:
            return custom.template_content
    template_content = getattr(cls, template_key, None)
    if template_content is None:
        logger.warning("未找到提示词模板: %s", template_key)
        raise ValueError(f"未找到提示词模板: {template_key}")
    return template_content
```

## 8. 约束与规范

### 8.1 接口兼容性

- `PromptService.get_template()` 方法签名不可变更
- 已有6个基础模板不可删除或修改 key 名称
- 新增模板必须使用类属性定义，与现有模式一致

### 8.2 模板格式

- 使用 Python `str.format()` 占位符语法：`{variable_name}`
- JSON 输出中的花括号必须双写：`{{` 和 `}}`
- 模板内容直接作为类属性赋值，不使用外部文件加载

### 8.3 安全要求

- 模板参数通过 `format_prompt()` 格式化，调用方需确保参数已清理
- 数据库自定义模板需校验 `template_content` 合法性
- 日志中禁止输出完整提示词内容
