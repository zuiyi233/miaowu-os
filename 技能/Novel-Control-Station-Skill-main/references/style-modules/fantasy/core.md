# 奇幻模块

奇幻最强的时候，不是设定被解释得很多，而是世界被活出来了。异质规则、能力体系、历史残留和日常生活必须互相咬合，读者才会信这个世界真的能运转。

## 适用时机

- 本章涉及虚构规则、魔法、血统、神器、制度或异世界常识
- 世界观需要通过社会纹理而非说明文显形
- 设定信息必须保持一致，又不能变成 lore dump
- 能力、资源和权力的代价要在场景里可见

## 覆盖子型

- 硬规则奇幻
- 软魔法、神话式、寓言式奇幻
- 高奇幻、低奇幻、宫廷与政治奇幻
- 冒险、黑暗、田园、都市与混合型奇幻

所有子型都适用本模块。通用不变项不是“必须公式化解释一切”，而是要有可感知的世界逻辑、代价逻辑和社会后果逻辑。

## 核心原则

- 规则要服务冲突，而不是取代冲突
- 能力越强，限制、代价、门槛和后果越要清楚
- 世界可信，不靠百科全书，靠日常细节、制度摩擦和人的选择
- 最好的设定信息总是随着人物需求一起进入，而不是停下来讲解
- 只展示本章需要承重的规则，其余设定可以留在背景
- 若采用软魔法或神话式处理，也要让读者感到其叙事边界、象征方向和风险，不可完全随心所欲

## 章节级操作

1. 先定义本章真正相关的一条世界规则。
2. 再决定它通过使用、摩擦、失败还是代价被看见。
3. 在需要世界厚度时，最好给一个生活化细节，让世界显得有人在其中长期生活。
4. 若出现新规则，立刻想清楚它会改写谁的行为和社会关系。
5. 若规则看似例外，尽量解释例外的来源和价格；若暂时不能全解释，至少标明它仍待确认的边界。

## 与其他风格混用

- 和 `suspense` 混用时，世界规则要让危险更具体，不要把设定当万能解答器
- 和 `romance` 混用时，制度、身份或能力成本可以直接变成关系障碍
- 和 `literary` 混用时，意象、历史和设定要共同承担主题，而不是彼此分家

## 修订检查

- 本章相关规则是否前后一致
- 力量出现时，代价和限制是否可见
- 主角之外的人是否也像真实世界居民一样生活
- 异常规则是否已经改变了行为、礼俗、机构或市场
- 有无整段设定说明可以改成场景化呈现

## 常见失误

- 设定倾倒
- 力量没有代价
- 世界只围着主角转
- 为了方便推进情节随手改规则
