# 恐怖模块

恐怖不是单次惊吓，而是让熟悉之物变得不再可靠，并且这种不可靠越来越难解释、难承受、难活过。恐惧应经过感知、推断和后果，而不是只经过形容词。

## 适用时机

- 不祥感需要跨章节或跨场景累积
- 日常现实需要被逐步扭斜，或现成威胁需要被不断具体化
- 受限视角、局部多视角或框架叙事能强化不确定和压迫
- 惊扰之后还需要留下余波

## 覆盖子型

- 超自然恐怖
- 心理恐怖
- 哥特、民俗、宗教、宇宙恐怖
- 身体恐怖、密闭求生、怪物型恐怖
- 与推理、悬疑、文学混合的恐怖

这些子型的外壳不同，但通用支架一致：正常秩序被污染，解释逐步失效，人物承受持续后果。受限视角是强工具，不是唯一合法写法。

## 核心原则

- 先让读者感觉不对，再决定何时解释
- 恐惧常常从普通事物的轻微错位开始，但明牌威胁、身体失控或空间困局也可以作为合法入口
- 感官证据越具体，恐惧越成立
- 视角越近，恐惧越私人；视角越稳，失真越有力
- 惊吓如果不改变人物、环境或规则，只会快速蒸发
- 场景、地点和氛围本身也能构成威胁，不要把恐怖全压给怪物出场

## 章节级操作

1. 先决定本章恐惧从哪里进入：日常错位、明牌威胁、身体失控、空间困局、知识污染，还是怪物逼近。
2. 再决定角色此刻能知道什么、误解什么、来不及知道什么。
3. 用与该入口匹配的证据把恐惧做实，例如感官异常、肉体变化、场所压迫、行为异样、时间错位或追逐痕迹。
4. 升级时优先增加解释失效、生存代价或污染范围，而不是单纯加大声响。
5. 收尾后留下一个持续存在的后果，让恐惧进入下一章。

## 与其他风格混用

- 和 `suspense` 混用时，悬念负责逼近，恐怖负责扭曲与余震
- 和 `mystery` 混用时，异常现象也要留下可追踪证据，不能只凭氛围
- 和 `literary` 混用时，意象可以加深压迫，但不要把可感的恐惧写成抽象寓言

## 修订检查

- 场景是否先扰动读者，再解释原因
- 恐惧是否通过感官证据成立
- 受限视角是否让不安更锋利，而不是让读者更糊涂
- 惊吓之后是否留下余波、损伤或规则变化

## 常见失误

- 解释来得太早
- 只说“很害怕”，不给证据
- 只有噪声，没有递进的不祥感
- 让怪物或答案完全代替氛围和场景
