# 幽默核心

小说里的幽默，最稳的引擎不是聪明，而是压力。角色越认真地想守住某件事，幽默就越容易成立。

## 常用引擎

- 角色拼命装稳，现实却持续拆台
- 自我认知与外部事实不匹配
- 地位翻转，强者暂时失手，弱者意外占上风
- 反复回避的问题终于无法回避
- 过度自信的解释撞上冷硬事实
- 一件小小失误被关系、场合或围观者放大

## 写作动作

- 给笑点绑定一个明确的人物弱点，例如面子、洁癖、傲慢、好胜、占有欲
- 先让角色认真行动，再让偏差显形；不要先想 punchline 再补场景
- 保留人物自己的说话方式，别为了搞笑把人写串味
- 让笑点和下一步行动相连，最好能逼出补救、掩饰或更大的失误

## 判断标准

- 读者会觉得意外，但回看又觉得必然
- 这个笑点换到另一个人物身上会变弱，因为它依赖此人的性格缺口
- 笑完之后，关系、地位或成本比之前更明确

## 优先选择

- 人物专属的幽默
- 关系里的幽默
- 场景式窘迫
- 压缩而具体的观察

## 避免

- 与场景任务无关的一次性俏皮话
- 用“有趣的叙述腔”淹没场景逻辑
- 靠羞辱弱者取笑却没有人物和情境支撑
- 用幽默抹掉本来该疼的情绪后果
