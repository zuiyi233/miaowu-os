# 幽默模块

把幽默当成叙事压力的转向器，不要当成悬空段子投递。真正有效的小说幽默，通常来自角色越想维持体面、权威、冷静或正确，现实越不给他面子。

## 适用时机

- 章节需要释放张力，但不能削弱 stakes
- 人物摩擦先要显得好笑，后面才更显得有代价
- 关系中的高低位、误判、逞强、遮掩需要被看见
- 对话节奏需要更利落、更有回弹

## 覆盖子型

- 角色窘迫型幽默
- 冷幽默与死正经反差
- 关系互怼与轻喜剧节奏
- 黑色幽默与苦中作笑
- 观察式、场景式、设定式幽默

以上子型都能调用本模块。通用不变项只有一条：笑意必须改变压力、关系、地位或读者对人物的理解。

## 核心原则

- 幽默最好从欲望、羞耻、偏见、嘴硬和误判里长出来
- 最稳的笑点不是“有趣的一句”，而是“角色认真时发生了错位”
- 笑点要么改变地位，要么暴露关系，要么制造后续成本
- 铺垫通常应更短，回报应更清楚，余波应比包袱更重要
- 同一个人不能在所有场景都同样好笑，幽默要有人物专属性
- 幽默也可以来自世界观察、场景安排和叙述角度，不必全部落在出丑

## 章节级操作

1. 先定义本章笑意主要从哪里来：窘迫、错位、观察、关系摩擦、设定反差，还是黑色余味。
2. 如果是人物或关系型幽默，再定义谁最怕丢脸、失控或被看穿，谁又最容易误判局势。
3. 如果是观察式、设定式或冷幽默，先明确哪一个日常逻辑、叙述角度或世界规则会产生反差。
4. 让笑意在最合适的时点改变场景，而不是让作者硬塞笑话。
5. 若使用回调，必须在更难堪、更公开、更失衡或更具后果的场合回返。
6. 如果本章还有严肃任务，笑点之后要留下关系、决定、理解偏差或代价的变化。

## 与其他风格混用

- 和 `romance` 混用时，优先让幽默暴露心动、逞强、误会和照顾，而不是破坏张力
- 和 `suspense` 混用时，幽默可来自错误自信或时机错位，但不能把危险感抹平
- 和 `literary` 混用时，幽默可以服务人物尊严与社会纹理，不要写成作者的机灵表演

## 修订检查

- 笑点是否来自人物处境，而不是作者跳出来抖机灵
- 去掉最搞笑的那一句后，场景是否仍然推进了关系或情节
- 铺垫是否足够短，回报是否足够清晰
- 回调是否带来了升级，而不是原样再说一遍
- 主要人物的笑法是否彼此不同

## 常见失误

- 把小说写成脱口秀
- 每个人都像同一个嘴贫角色
- 为了笑点破坏人物情绪连续性
- 只把场面写得更吵，却没有让代价更高
