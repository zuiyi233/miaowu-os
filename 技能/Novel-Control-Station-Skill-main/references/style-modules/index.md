# 内部风格模块索引

用这个索引在 `novel-control-station` 内部做风格路由，不要凭印象混读全部模块。

## 总规则

- 只加载当前章节真正激活的风格模块
- 永远先读对应模块的 `core.md`
- 只有当章节压力已经明确时，才继续读取更深文件
- 同时保持最多 2 个主风格和 1 个辅助风格
- 风格模块服务于章节任务，不要把它们当成装饰性气味包

## 选择顺序

1. 先判断章节主要靠什么推进。
2. 再判断读者此章最该感到什么。
3. 最后判断哪一个辅助风格能增强，不是稀释，当前主线压力。

如果一个模块只是让文本“更像某种类型”而不能改变压力、关系、风险或意义，就不要加载它。

## Humor / 幽默

- 适用于章节需要由窘迫、失衡、观察反差、冷幽默、回调或说话节奏制造笑意
- 先读：`humor/core.md`
- `humor/humor-core.md`
- `humor/dialogue-comedy.md`
- `humor/callback-and-escalation.md`

## Suspense / 悬念

- 适用于章节需要维持未知、危险、暴露风险、延迟揭示或强前拉钩子
- 先读：`suspense/core.md`
- `suspense/suspense-core.md`
- `suspense/chapter-hook.md`

## Mystery / 推理

- 适用于项目需要可调查问题、公平线索、可能性分支、误导和分段揭晓
- 先读：`mystery/core.md`
- `mystery/clue-and-misdirection.md`
- `mystery/reveal-order.md`

## Romance / 爱情

- 适用于章节核心由吸引、阻碍、情感债、未决亲密或关系温度变化驱动
- 先读：`romance/core.md`
- `romance/emotional-debt.md`
- `romance/subtext-and-tension.md`

## Horror / 恐怖

- 适用于章节核心由不祥感、明牌威胁、身体或空间压迫、感官惊扰、视角限制和余波构成
- 先读：`horror/core.md`
- `horror/fear-escalation.md`
- `horror/limited-pov.md`

## Fantasy / 奇幻

- 适用于章节需要处理虚构规则、神话性介入、权力代价、制度纹理、异世界日常和设定一致性
- 先读：`fantasy/core.md`
- `fantasy/world-rules-and-cost.md`
- `fantasy/everyday-world-thickness.md`

## Literary / 文学

- 适用于章节需要强化主题压力、意象回返、语言纪律、结构意味和解释层次
- 先读：`literary/core.md`
- `literary/theme-pressure.md`
- `literary/language-health.md`

## 混用提醒

- `suspense + mystery`：前者管悬而未决的压力，后者管证据与答案的公平性
- `romance + humor`：优先让笑点暴露亲密风险或关系差值，不要把情绪债轻易消解掉
- `horror + suspense`：恐怖更重失真和余波，悬念更重未知与逼近，二者通常能叠加，但不要混成一团
- `fantasy + literary`：最好让设定与语言都承担意义，避免一个只负责说明、一个只负责装饰
