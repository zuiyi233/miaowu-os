# 文学模块

文学性不是高腔调，而是语言、人物和主题在场景里互相施压。深度必须通过选择、后果、意象回返和结构张力生长出来，而不是靠作者直接宣布意义。

## 适用时机

- 章节需要强化主题压力
- 某个意象、母题、场景回声或形式安排需要承担意义
- 语言纪律、叙述密度、心理深描或结构性对照需要提升
- 项目明确要求偏文学、克制、实验或高解释层次

## 覆盖子型

- 现实主义文学
- 抒情、冷峻、极简、繁密、碎片化写法
- 心理深描、社会观察、形式实验
- 与悬疑、奇幻、爱情等类型混合的文学化写法

本模块不是在规定单一文风。通用不变项只有这些：形式要承担意义，语言要承重，人物与主题要互相施压。

## 核心原则

- 主题应压在人物身上，不应飘在人物上方
- 意象和母题要随着情境变化而变义，不要只做漂亮回环
- 语言的精确度比语言的华丽度重要
- 结构复杂只有在揭示人物真相、时间关系或意义冲突时才值得使用
- 含混可以有，但必须是多义，不是糊
- 文风可以繁、可以简，但都必须让读者更接近经验，而不是更接近作者姿态

## 章节级操作

1. 先写清本章到底在逼问什么价值冲突。
2. 再决定哪一个动作、意象或结构回返来承重。
3. 把抽象命题拆回人物选择和后果。
4. 修辞出现时问一句：它除了好看，还做了什么场景工作。
5. 如使用跳时、重叠视角或非线性，应让它揭示新的心理或意义层；若只是调度便利，通常不值得上这种结构。

## 与其他风格混用

- 和 `romance` 混用时，主题不能替代互动，必须落到关系选择
- 和 `mystery` 混用时，象征和含义不能破坏线索清晰度
- 和 `fantasy` 混用时，世界观细节与主题意象应相互支撑

## 修订检查

- 章节是否真的压出了一个价值冲突
- 意象是否承重，而非装饰
- 语言是否活着，而不是膨胀着
- 结构是否揭示人物真相或主题关系
- 评论是否已经取代了戏

## 常见失误

- 用抽象句制造假深度
- 语言死、空、肿
- 非线性只是花样
- 作者评论挤掉人物经验
