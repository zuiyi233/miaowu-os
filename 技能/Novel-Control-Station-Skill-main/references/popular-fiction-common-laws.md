# Popular Fiction Common Laws

Use this when a project aims at high readability, strong discussion value, market traction, or dual-high balance.

These are benchmark laws distilled from high-heat and high-reputation fiction samples. They are reference pressures, not templates to imitate.

## 1. Theme Must Land In Action

- Theme is valid only if it changes a decision, relationship, or cost.
- Reject outline language that sounds deep but does not force anyone to choose.

## 2. Characters Need Pressure And Motion

For each core character, define:

- what they want
- what blocks them
- what the pursuit costs
- how they react when pressure rises
- what arc movement becomes possible this chapter

Flat role labels are not enough.

## 3. Chapters Need Hook, Propulsion, Closure, Debt

Every chapter should know:

- what opens curiosity or tension
- what pushes the story forward
- what closes locally
- what debt carries forward

If the chapter only "contains events," it is structurally weak.

## 4. Multi-Line Interference Beats Parallel Display

- Lines must collide, contaminate, or reroute one another.
- A side line that does not change the main line is decoration.

## 5. Readability Is Not Simplicity

- Surface readability must stay clear even when the structure is layered.
- Information should arrive in the order that maximizes tension and comprehension together.

## 6. Memory Points Matter

- Each chapter should leave at least one memorable move:
  - a charged image
  - a value collision
  - a reversal with consequence
  - a line of action that changes how a character is seen

## 7. Arc Progress Requires Increment

- Character arcs do not move only at major plot points.
- Track small but real increments:
  - new fear
  - new courage
  - new debt
  - new self-knowledge
  - new relation status

## 8. Hook Density Must Match Mode

- Heat-oriented work can push harder on cliff severity and episode pull.
- Reputation-oriented or slow-burn work may soften hook density.
- In every case, confusion, drag, and emotional flatness still fail.

## 9. Originality Outranks Benchmark Worship

- Extract principles, not shells.
- If the draft feels like "this is the X version of Y," stop and rebuild the pressure system.

## 10. Compliance Is Part Of Craft

Never imitate:

- signature plot engines
- famous reveal orders
- iconic cast shells
- distinctive language texture

Turn every borrowed lesson into a project-specific constraint.

## Failure Signals

- slogan-like theme talk
- characters reacting as labels, not people
- chapter has movement but no closure
- chapter has closure but no carryover debt
- lots of incident, little residue
- recognizable benchmark cosplay

## Adaptive Note

Benchmark logic is a calibrated reference system.

It must adapt to:

- user intent
- genre cluster
- audience promise
- pacing mode

It must not overwrite them.
