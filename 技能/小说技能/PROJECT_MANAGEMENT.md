# 长篇小说项目管理系统

为解决长篇小说创作中常见的前后矛盾、伏笔丢失、人物遗忘等问题，使用以下基于Markdown文件的管理系统。所有文件固定存放在目标小说项目目录下的`.novel`目录中。

## .novel目录结构

### 基础结构（适合中短篇小说）
```
小说项目/
├── .novel/
│   ├── characters.md          # 人物档案
│   ├── timeline.md            # 时间线记录
│   ├── foreshadowing.md       # 伏笔追踪
│   ├── settings.md            # 世界观设定
│   ├── outline.md             # 大纲与章节规划
│   └── chapter_logs.md        # 章节日志
└── volumes/                   # 小说正文目录（按卷分组）
    ├── volume1/               # 第一卷
    │   ├── chapter001.md
    │   ├── chapter002.md
    │   └── ...
    ├── volume2/               # 第二卷
    │   ├── chapter101.md
    │   ├── chapter102.md
    │   └── ...
    └── ...
```

```
│   ├── style/                 # 风格管理目录
│   │   ├── index.md           # 风格总览
│   │   ├── expression.md      # 表达指纹
│   │   └── corpus.md          # 个人语料库
│   │
│   ├── outline/               # 大纲目录
│   │   ├── index.md           # 大纲总览
│   │   ├── main.md            # 主线大纲

│   │       └── volume2.md
│   │
│   └── logs/                  # 日志目录
│       ├── index.md           # 进度总览
│       └── chapters/          # 章节日志（按卷分组）

    ├── volume1/               # 第一卷
    │   ├── chapter001.md
    │   ├── chapter002.md
    │   └── ...
    ├── volume2/               # 第二卷
    │   ├── chapter101.md
    │   ├── chapter102.md
    │   └── ...
    └── ...
```

### 目录选择建议
- **50万字以下**：使用基础结构
- **50-100万字**：可选择基础结构或扩展结构
- **100万字以上**：强烈推荐使用扩展结构

## 工作流程建议

### 使用基础结构的工作流程

1. **开始新项目**：
   - 创建.novel目录和所有模板文件
   - 创建volumes目录用于存储小说正文
   - 在volumes目录下创建volume1目录开始第一卷的创作
   - 先完成settings.md和outline.md
   - 设计主要人物，填写characters.md

### 使用扩展结构的工作流程

1. **开始新项目**：
   - 创建.novel目录及所有子目录
   - 创建volumes目录用于存储小说正文
   - 在volumes目录下创建volume1目录开始第一卷的创作
   - 创建index.md项目总索引
   - 先完成settings/目录下的基础设定文件
   - 完成outline/目录下的主线大纲
   - 设计主要人物，为每个主要人物创建单独的md文件

2. **写作前**：
   - 查阅index.md获取项目概览
   - 查阅timeline/index.md和当前卷时间线
   - 查阅characters/index.md找到需要的人物
   - 查阅foreshadowing/active.md看是否有伏笔需要回收
   - 查阅settings/目录下相关设定文件

3. **写作中**：
   - 遇到新主要人物：在characters/main/创建单独文件，并更新characters/index.md
   - 遇到新次要人物：在characters/secondary/创建单独文件，并更新characters/index.md
   - 遇到新配角：记录到对应卷的characters/supporting/volumeX.md
   - 设置新伏笔：记录到foreshadowing/active.md，分配唯一ID，并更新foreshadowing/index.md
   - 回收伏笔：将伏笔从foreshadowing/active.md移动到foreshadowing/resolved.md，并更新索引
   - 重要时间点：记录到timeline/volumes/对应卷文件
   - 新设定：记录到settings/目录下对应主题文件，并更新settings/index.md

4. **写作后**：
   - 更新timeline/volumes/对应卷的章节时间对照表
   - 填写logs/chapters/对应卷的章节日志
   - 更新logs/index.md的进度统计
   - 更新outline/目录下对应卷的章节规划
   - 更新index.md的最近更新和当前状态
   - 检查是否有前后矛盾

5. **定期检查**：
   - 每周检查一次index.md的重要提醒清单
   - 检查foreshadowing/active.md中是否有逾期未回收的伏笔
   - 确保characters/目录下人物状态一致性
   - 检查timeline/目录下时间线逻辑是否一致
   - 清理过期或无用的信息
   - 更新各index.md文件的统计数据
