## 2. 修复方案与验收标准（已落地）

> 说明：下表只列“确认问题”（含复核新增且高优先级的问题）。

| 问题ID | 确认结论 | 修复策略 | 关键文件 | 验收标准 |
|---|---|---|---|---|
| A5-01 | 前端弱随机回退（安全） | 移除 `Math.random()` fallback；不支持 WebCrypto 时直接报错 | `frontend/src/core/ai/crypto.ts` | 代码中不再出现弱随机回退路径 |
| A5-02 | 密钥指纹输出到控制台 | 删除指纹输出；`validateEncryptionConfig` 返回 `hidden` | `frontend/src/core/ai/crypto.ts` | 不再输出 key fingerprint |
| A5-04 | `/api/ai/*` 缺少访问控制 | 增加 token/loopback 双策略 + 限流；保留 401/403/429 语义 | `backend/app/gateway/api/ai_provider.py` | 未带 token 的远程访问被拒；超频返回 429 |
| A5-05 | 请求体携带明文 api_key | 前端停止发送 api_key；后端忽略并告警 deprecated 字段 | `frontend/src/core/ai/global-ai-service.ts`、`backend/app/gateway/api/ai_provider.py` | 抓包或代码审阅不再有 `provider_config.api_key` 出站 |
| C8-01 | novel_stream 缺少频率限制 | 为高成本流式/分析接口加 per-user+action 限流 | `backend/app/gateway/novel_migrated/api/novel_stream.py` | 超频请求返回 429 |
| C8-02 | 分析缓存无清理策略 | 增加 TTL + 最大条目清理，读取前清理 | `backend/app/gateway/novel_migrated/api/novel_stream.py` | 过期/溢出条目会被回收 |
| U4-01(修正后) | key 校验时机与启动失败语义不清晰 | 增加 `validate_encryption_key`，在启动期 fail-fast 非法 key | `backend/app/gateway/novel_migrated/core/crypto.py`、`backend/app/gateway/app.py` | 非法 key 启动即报错；合法 key 正常启用 |
| N1（复核新增） | retry 参数耦合 | 拆分 request timeout 与 retry delay | `frontend/src/core/ai/global-ai-service.ts` | 超时与退避参数独立可控 |
| N2（复核新增） | SSE 分片解析不稳 | 增加 stream 读取超时 + buffer 拼包 | `frontend/src/core/ai/global-ai-service.ts` | 分片 data 行可连续解析，不丢尾包 |
| N3（复核新增） | 解密失败静默回退掩盖损坏 | 对“看似加密但解密失败”返回空值并提示重录；仅兼容历史明文 | `frontend/src/core/ai/crypto.ts` | 损坏密文不会伪装成“可用 key” |
