# 小说功能完整迁移方案（Miaowu-OS）

> 目标：将参考项目（MuMuAINovel-main）的小说功能完整迁移至主项目（miaowu-os），保留主项目完善的AI能力，复现参考项目清晰的交互逻辑与路由架构。
> 
> 排除范围：账户功能（登录/注册/用户管理）不做参考。

---

## 0. 核验说明（2026-04-18）

本方案已基于以下代码进行交叉核验：
- 主项目（二开）：`/mnt/d/miaowu-os/deer-flow-main`
- 原版 Deer-Flow：`/mnt/d/deer-flow-main`
- 小说参考项目：`/mnt/d/miaowu-os/参考项目/MuMuAINovel-main`

核验结论摘要（仅列与实施阻塞相关项）：
- 主项目前端 `frontend/src/core/novel/novel-api.ts` 仅有小说/章节/实体基础 CRUD 与部分扩展接口，**没有** `generateChapter/continueChapter/polishChapter/batchGenerateChapters` 这类章节 AI 方法。
- 主项目 `frontend/src/core/novel/useNovelStore.ts` 目前只有 `setChapters/setNovels` 等 setter，**没有** `add*/update*/remove*` 的细粒度数组操作。
- 主项目后端当前可用小说路由主要在 `backend/app/gateway/routers/novel.py`（基础 CRUD）；`novel_migrated` 仅落地了 `wizard_stream/memories/careers/foreshadows` 等模块，**尚未落地** 章节/大纲/角色完整 AI 路由。
- 参考项目章节流式生成路径是 `POST /api/chapters/{chapter_id}/generate-stream`（见 `MuMuAINovel-main/frontend/src/store/hooks.ts`），并非文档原写的 `/api/novels/{novel_id}/sse/*`。
- 参考项目 `Chapter` 模型核心字段为 `project_id/chapter_number/status(draft|writing|completed)/expansion_plan(string)`，与本方案旧版本定义存在偏差。
- 原版 Deer-Flow（`/mnt/d/deer-flow-main`）不包含小说模块，对本次“小说功能迁移”只能作为网关/工程结构参考，不作为小说业务 API 的对照基线。

## 1. 现状分析

### 1.1 主项目（miaowu-os）当前架构

#### 1.1.1 路由结构（Next.js App Router）
```
/workspace/novel                          # 书架首页 (NovelHome)
/workspace/novel/[novelId]                # 工作区容器 (NovelWorkspace)

/workspace/novel/inspiration              # 灵感模式 (InspirationMode)
/workspace/novel/book-import              # 拆书导入 (BookImportPage)
```

#### 1.1.2 工作区视图模式（单页面内切换）
NovelWorkspace 通过 `viewMode` 状态切换以下视图：
- `editor` - 编辑器
- `reader` - 阅读器
- `outline` - 大纲管理
- `timeline` - 时间线
- `graph` - 关系图
- `careers` - 职业体系
- `foreshadows` - 伏笔管理
- `settings` - 设置

#### 1.1.3 存在的问题
1. **URL 无法直接定位到具体功能** - 所有功能在同一个页面内切换，无法通过URL分享特定功能页面
2. **功能分散且入口不统一** - 角色、组织、物品等管理分散在侧边栏弹窗中，没有独立页面
3. **章节管理体验不佳** - 没有独立的章节列表管理页，所有操作在编辑器内完成
4. **缺少剧情分析独立页** - 分析功能仅作为弹窗存在
5. **写作风格/提示词工坊入口过深** - 作为设置子项，不易发现

### 3.3 阶段三：AI能力整合（2-3天）

#### 3.3.1 复现参考项目AI调用逻辑

**核心原则**: 复用主项目现有网关与 `novelApiService` 调用风格，采用 `fetch + ReadableStream` 处理流式响应（与参考项目一致，不使用原生 `EventSource` 的 POST 模式）。

**SSE客户端封装**:
```typescript
// deer-flow-main/frontend/src/core/novel/sse-client.ts
// 参考参考项目的 SSEPostClient，但使用主项目的 AI 服务

export class NovelSSEClient {
  constructor(private novelId: string) {}
  
  async generateChapter(chapterId: string, params: GenerateChapterParams, callbacks: SSECallbacks) {
    const response = await fetch(
      buildUrl(`/novels/${this.novelId}/chapters/${chapterId}/generate-stream`),
      {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(params),
      }
    );
    // 通过 parseSseStream 逐帧解析，避免一次性读入
    await this.consumeStream(response, callbacks);
  }
  
  async continueChapter(chapterId: string, params: ContinueChapterParams, callbacks: SSECallbacks) {
    const response = await fetch(
      buildUrl(`/novels/${this.novelId}/chapters/${chapterId}/continue-stream`),
      {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(params),
      }
    );
    await this.consumeStream(response, callbacks);
  }
  
  async batchGenerate(params: BatchGenerateParams, callbacks: SSECallbacks) {
    const response = await fetch(
      buildUrl(`/novels/${this.novelId}/chapters/batch-generate-stream`),
      {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(params),
      }
    );
    await this.consumeStream(response, callbacks);
  }
  
  // ... 大纲/角色/世界观流式操作
}
```

### 3.4 阶段四：数据流对齐（1-2天）

#### 3.4.1 同步 Hooks 实现
参考参考项目的 `store/hooks.ts`，为主项目创建对应的同步 hooks。实现时使用 `novelApiService`（非 `novelApi`）：

```typescript
// deer-flow-main/frontend/src/core/novel/sync-hooks.ts

export function useCharacterSync(novelId: string) {
  const { setCharacters, addCharacter, updateCharacter, removeCharacter } = useNovelStore();
  
  const refreshCharacters = async () => {
    const novel = await novelApiService.getNovelByIdOrTitle(novelId);
    const characters = novel?.characters ?? [];
    setCharacters(characters);
    return characters;
  };
  
  const createCharacter = async (data: Character) => {
    const result = await novelApiService.createCharacter(novelId, data);
    addCharacter(result);
    return result;
  };
  
  const updateCharacterSync = async (entity: Character) => {
    const result = await novelApiService.updateCharacter(entity);
    updateCharacter(result.id, result);
    return result;
  };

  const deleteCharacter = async (id: string) => {
    // 需补充 novelApiService.deleteCharacter 后再接入
    removeCharacter(id);
  };
  
  return { refreshCharacters, createCharacter, updateCharacter: updateCharacterSync, deleteCharacter };
}

```

#### 3.4.2 TanStack Query 整合
保留现有 Query hooks，但修正为“**服务端状态单一事实源**”：
- TanStack Query：管理服务端状态（章节正文、列表、详情）。
- Zustand：只管理 UI 状态（当前章节ID、侧边栏折叠、生成任务进度等）。
- 避免把同一份远端数据长期同时存入 Query 和 Store，防止双写不一致与重复渲染。

```typescript
// mutation 成功后优先更新 query cache；store仅存UI态
export function useCharacterSync(novelId: string) {
  const queryClient = useQueryClient();
  const { addCharacter } = useNovelStore();
  
  const createCharacter = async (data: Character) => {
    const result = await novelApiService.createCharacter(novelId, data);
    queryClient.setQueryData(['characters', novelId], (prev: Character[] = []) => [...prev, result]);
    // 若本页依赖store立即渲染，可做轻量同步
    addCharacter(result);
    return result;
  };

  const refresh = () => queryClient.invalidateQueries({ queryKey: ['characters', novelId] });

  return { createCharacter, refresh };
}
```

### 10.2 SSE客户端实现参考

```typescript
// deer-flow-main/frontend/src/core/novel/sse-client.ts

export interface SSECallbacks<T> {
  onProgress?: (message: string, progress: number) => void;
  onChunk?: (chunk: string) => void;
  onResult?: (result: T) => void;
  onError?: (error: string) => void;
  onComplete?: () => void;
}

export class NovelSSEClient {
  private baseUrl: string;
  
  constructor(private novelId: string) {
    this.baseUrl = getBackendBaseURL();
  }
  
  async generateChapter(
    chapterId: string,
    params: GenerateChapterParams,
    callbacks: SSECallbacks<Chapter>
  ): Promise<void> {
    return this.connectSSE(
      `/api/novels/${this.novelId}/chapters/${chapterId}/generate-stream`,
      params,
      callbacks
    );
  }
  
  private async connectSSE<T>(
    endpoint: string,
    params: unknown,
    callbacks: SSECallbacks<T>
  ): Promise<void> {
    const response = await fetch(`${this.baseUrl}${endpoint}`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(params),
    });
    
    const reader = response.body?.getReader();
    if (!reader) throw new Error('No response body');
    
    const decoder = new TextDecoder();
    let buffer = '';
    
    try {
      while (true) {
        const { done, value } = await reader.read();
        if (done) break;
        
        buffer += decoder.decode(value, { stream: true });
        // 以 SSE 帧边界分割，避免 chunk 切分导致 JSON 半包
        const lines = buffer.split('\n\n');
        buffer = lines.pop() || '';
        
        for (const line of lines) {
          const dataMatch = line.match(/^data:\s*(.+)$/m);
          if (dataMatch) {
            const data = JSON.parse(dataMatch[1]);
            this.handleSSEEvent(data, callbacks);
          }
        }
      }
      
      callbacks.onComplete?.();
    } catch (error) {
      callbacks.onError?.(String(error));
    }
  }
  
  private handleSSEEvent<T>(data: any, callbacks: SSECallbacks<T>): void {
    switch (data.type) {
      case 'progress':
        callbacks.onProgress?.(data.message, data.progress);
        break;
      case 'chunk':
        callbacks.onChunk?.(data.content);
        break;
      case 'result':
        callbacks.onResult?.(data.payload);
        break;
      case 'error':
        callbacks.onError?.(data.message);
        break;
    }
  }
}
```

```typescript
// deer-flow-main/frontend/src/core/novel/sync-hooks.ts

import { useQueryClient } from '@tanstack/react-query';
import { useNovelStore } from './useNovelStore';
import { novelApiService } from './novel-api';

export function useChapterSync(novelId: string) {
  const queryClient = useQueryClient();
  const store = useNovelStore();
  
  const refreshChapters = async () => {
    const novel = await novelApiService.getNovelByIdOrTitle(novelId);
    const chapters = novel?.chapters ?? [];
    store.setChapters(chapters);
    return chapters;
  };
  
  const createChapter = async (data: ChapterCreate) => {
    const chapter = await novelApiService.createChapter(novelId, data as unknown as Chapter);
    store.addChapter(chapter);
    queryClient.invalidateQueries({ queryKey: ['novel', novelId] });
    return chapter;
  };
  
  const updateChapter = async (chapterId: string, data: ChapterUpdate) => {
    const chapter = await novelApiService.updateChapter(novelId, chapterId, data as Partial<Chapter>);
    store.updateChapter(chapterId, chapter);
    queryClient.invalidateQueries({ queryKey: ['novel', novelId] });
    return chapter;
  };
  
  const deleteChapter = async (chapterId: string) => {
    await novelApiService.deleteChapter(novelId, chapterId);
    store.removeChapter(chapterId);
    queryClient.invalidateQueries({ queryKey: ['novel', novelId] });
  };
  
  const setCurrentChapter = (chapter: Chapter | null) => {
    store.setCurrentChapter(chapter);
  };
  
  return {
    chapters: store.chapters,
    currentChapter: store.currentChapter,
    refreshChapters,
    createChapter,
    updateChapter,
    deleteChapter,
    setCurrentChapter,
  };
}
```

## 12. 附加：一轮性能优化建议（联网核验）

本节为可选增强项，来源于 2026-04-18 联网调研（FastAPI/Starlette/MDN/TanStack/React/Zustand 官方文档）。

### 12.1 P0（建议随迁移主线一并落地）

1. 流协议标准化  
统一事件字段：`event/type/id/seq/chapterId/data/done/error`，服务端固定 `text/event-stream`。
2. 断线恢复机制  
支持 `id + Last-Event-ID` 或业务 `cursor`，避免断线后整章重跑。
3. 降低渲染频率  
前端按时间窗（如 `requestAnimationFrame`）合并 chunk，避免 token 级 `setState`。
4. 状态边界收敛  
Query 管服务端真相，Store 仅存 UI 态（任务ID、面板状态、选中章节、滚动位置）。
5. 取消与超时  
统一 `AbortController`，切章/离页/超时时中止请求并回收 reader。
6. 代理与网关配置  
Nginx 关闭缓冲（`proxy_buffering off`），配置心跳（10-15s）避免上游空闲超时。

### 12.2 P1（后续增强）

1. 可观测指标  
记录每章：chunk 间隔、重连次数、解析失败率、平均恢复耗时、渲染次数。
2. Worker 化重活  
大文本增量清洗、分段、关键词提取移到 Web Worker，降低主线程阻塞。
3. 批量生成协议细化  
`batch-start/item-start/item-delta/item-done/batch-done`，每个 item 独立状态。

### 12.3 实施提示

- 原生 `EventSource` 不适合 POST 场景；本项目继续采用 `fetch + ReadableStream`。
- 若后续要切换到 `POST 创建任务 + GET 订阅SSE`，需同时调整鉴权和任务状态管理。
