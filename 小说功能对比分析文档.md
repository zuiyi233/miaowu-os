## 3. 当前项目实现事实（deer-flow-main）

### 3.1 路由与工作区

- 路由页面：
  - `frontend/src/app/workspace/novel/page.tsx`
  - `frontend/src/app/workspace/novel/[novelId]/page.tsx`
  - `frontend/src/app/workspace/novel/[novelId]/editor/page.tsx`
- 工作区容器：`frontend/src/components/novel/NovelWorkspace.tsx`
- 视图模式状态枚举：`useNovelStore` 中定义 `home/editor/reader/graph/timeline/chat/outline/settings`
- `NovelWorkspace` 主渲染分支可见：`editor/reader/outline/timeline/graph/settings`；`chat` 当前在 `editor` 分支以内右侧 `AiPanel`（桌面）与 `Sheet`（移动端）呈现

### 3.2 本地数据层（Dexie）

- 数据库：`frontend/src/core/novel/database.ts`
- `version(1)` 基础表：
  - `novels`、`volumes`、`chapters`、`characters`、`settings`、`factions`、`items`
  - `promptTemplates`、`relationships`、`timelineEvents`、`graphLayouts`
  - `snapshots`、`chatSessions`
- `version(2)` 增量表（reader 相关）：
  - `annotationThreads`、`recommendationItems`、`auditLog`
- 可见能力：
  - 小说数据：`saveNovel/loadNovel/getAllNovels/getDashboardStats`
  - 导入导出：`exportAllData/importData/clearAllData`
  - reader 数据：批注线程、建议项、审计日志读写接口

### 3.3 角色、势力、关系、物品

- 数据模型：`frontend/src/core/novel/schemas.ts`（`characterSchema`、`factionSchema`、`relationshipSchema`、`itemSchema`）
- UI 组件：`EntitySidebar`、`EntityManager`、`RelationshipManager`、`RelationshipGraph`
- 可见操作：本地 CRUD、关系图可视化、实体选择器与 Mention 组件

### 3.4 大纲

- 组件：`OutlineView`、`OutlineListPanel`、`OutlineItem`、`MiniOutlineView`、`OutlineConfigPanel`
- 状态：`useOutlineStore`（树结构、选择、重排、生成中状态）
- AI 服务层存在 `generateOutline()`（`frontend/src/core/novel/ai-service.ts`），代码可见到前端调用链

### 3.5 章节编辑与历史快照

- 编辑器：`frontend/src/components/novel/Editor.tsx`（TipTap）
- 工具栏：`EditorToolbar.tsx`（续写、润色、扩写、压缩、改写、实体分析按钮）
- 快照：`createSnapshot/getChapterSnapshots/getLatestSnapshot`
- 历史面板：`HistorySheet.tsx`

### 3.6 AI 功能（前端调用链）

- 服务：`frontend/src/core/novel/ai-service.ts`
- 可见方法：
  - `chat()`
  - `generateOutline()`
  - `continueWriting()`
  - `polishText()`
  - `expandScene()`
  - `condenseText()`
  - `rewriteText()`
  - `extractEntities()`
- 端点调用逻辑：
  - 传入 `novelId` 时：`/api/novels/{novelId}/ai/chat`
  - 未传 `novelId` 时：`/api/novel/chat`（兼容路径）
- `frontend/src/core/novel/novel-api.ts` 同样存在 `chat(novelId, payload)` 对 `/api/novels/{novelId}/ai/chat` 的调用

### 3.7 时间线

- 模型：`timelineEventSchema`
- 组件：`TimelineView`、`TimelineEventForm`
- 查询与变更：`useTimelineEventsQuery/useAddTimelineEventMutation/useDeleteTimelineEventMutation`
- 网关后端可见同域时间线接口：`GET/POST /api/novels/{novel_id}/timeline`、`PUT/DELETE /api/novels/{novel_id}/timeline/{event_id}`

### 3.8 提示词、设置、文风

- 模型：`promptTemplateSchema`（类型包含 `outline/continue/polish/expand/chat/extraction`）
- 数据层：`addPromptTemplate/updatePromptTemplate/deletePromptTemplate/setActivePromptTemplate`
- 设置 UI（当前挂载）：`NovelSettings` 页面包含 `PromptTemplateManager` 与 `DataManagement` 两个页签
- 本地状态：`useSettingsStore`、`useStyleStore`
- 说明：`ProviderSettings.tsx` 文件存在，但当前 `NovelSettings` 页面未挂载该组件

### 3.9 Reader 审阅工作台

- 主组件：`frontend/src/components/novel/reader/ReaderWorkspaceView.tsx`
- 可见子页签：`reading`、`diff`、`structure`、`recommendation`、`annotation`、`quality`、`audit`
- 对应子组件：`ReadingMode`、`VersionDiff`、`StructureLinkedView`、`RecommendationPanel`、`AnnotationThreadPanel`、`QualityReportPanel`、`AuditLogPanel`
- 可见调用链：
  - 质量报告：`/api/novels/{novel_id}/quality-report`
  - 审计日志：`/api/novels/{novel_id}/audits`
  - 推荐：`/api/novels/{novel_id}/recommendations`、`/generate`、`/{rec_id}/accept`
  - 本地回退：`auditLog`、`annotationThreads`、`recommendationItems`

### 3.10 小说后端网关路由（当前项目）

- 路由文件：`backend/app/gateway/routers/novel.py`
- 注册方式：`backend/app/gateway/app.py` 中 `CORE_ROUTER_MODULES = ("app.gateway.routers.novel",)` 并统一 `include_router`
- 路由前缀：`/api`
- 主路径端点分组（可见）：
  - novel：列表/详情/创建/更新/删除
  - chapter：列表/详情/创建/更新/删除
  - entity：列表/创建/更新/删除
  - timeline：列表/创建/更新/删除
  - graph：读取/保存
  - recommendation：读取/生成/采纳
  - interaction：列表/创建/更新/删除
  - quality report：读取
  - audits：读取（支持筛选参数）
  - ai chat：`POST /api/novels/{novel_id}/ai/chat`（兼容 `POST /api/novel/chat`）
- 存储实现：`NovelStore` 内存字典 + 本地 JSON 文件 `backend/app/gateway/data/novel_store.json`
- Chat 返回形态：非流式 JSON 与 SSE 流式（`message/done/error`）

### 3.11 当前项目代码可见边界

- 已定位到小说前端实现 + 网关后端路由实现；“仅前端预留”结论不再成立。
- 后端存储当前为 `NovelStore` + JSON 文件落盘，未定位到小说域数据库 ORM 持久化实现。
- 推荐生成与质量报告均为规则/占位逻辑，未定位到向量检索、章节记忆分析或伏笔状态机链路。
- 未定位到小说域独立职业体系（`Career/CharacterCareer`）、伏笔模型、`PlotAnalysis/StoryMemory` 等参考项目同层实现。

## 5. 差异汇总（事实）

### 5.1 参考项目已定位、当前项目未定位到对应实现链路的能力

- 职业体系（`Career` / `CharacterCareer`）
- 伏笔独立模型与状态机（含 `partially_resolved`、`abandoned`）
- 章节分析持久化（`PlotAnalysis`）与记忆持久化（`StoryMemory`）
- 向量数据库与服务侧语义检索（ChromaDB + SentenceTransformer）
- 小说域 MCP 工具加载与多轮工具调用链
- 拆书导入流水线与导入后自动生成链路
- 封面生成、灵感生成独立 API
- 小说域独立认证/后台链路

### 5.2 当前项目已定位能力

- 小说工作区多视图：`editor/reader/outline/timeline/graph/settings`
- Reader 审阅工作台：阅读、版本对比、结构联动、建议、批注、质量、审计
- 本地 Dexie 多表管理（含 `annotationThreads/recommendationItems/auditLog`）
