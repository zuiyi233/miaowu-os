### 6.3 get_template 核心方法改造 ❌ **未完成（关键阻塞项）**

**当前状态**：[prompt_service.py:1722-1734](file:///D:/miaowu-os/deer-flow-main/backend/app/gateway/novel_migrated/services/prompt_service.py#L1722-L1734)

```python

async def get_template(
    cls,

    user_id: str,

) -> str:
```

**问题分析**：
1. ❌ 注释明确说明"当前无自定义模板表，直接返回内置模板" — **信息过时**
2. ❌ `del user_id, db` — 直接丢弃了数据库会话和用户ID参数
3. ❌ 没有查询 `prompt_templates` 表的逻辑
4. ❌ 不支持用户自定义模板覆盖系统默认的功能

**预期改造方案**（参考报告第7.2节）：

```python
@classmethod
async def get_template(
    cls,
    template_key: str,
    user_id: str,
    db,
) -> str:
    """获取模板（优先返回用户自定义模板，否则返回系统默认）。"""
    # 1. 尝试从数据库获取用户自定义模板
    if user_id and db:
        try:
            result = await db.execute(
                select(PromptTemplate).where(
                    PromptTemplate.user_id == user_id,
                    PromptTemplate.template_key == template_key,
                    PromptTemplate.is_active == True
                )
            )
            custom = result.scalar_one_or_none()
            if custom and custom.template_content:
                return custom.template_content
        except Exception as e:
            logger.warning(f"查询自定义模板失败: {e}, 使用系统默认")

    # 2. 回退到系统内置模板
    template_content = getattr(cls, template_key, None)
    if template_content is None:
        logger.warning("未找到提示词模板: %s", template_key)
        raise ValueError(f"未找到提示词模板: {template_key}")
    return template_content
```

**影响评估**：
- 🔴 **高影响**：虽然API和管理界面都已就绪，但核心读取逻辑未打通
- 🔴 用户通过API创建的自定义模板无法被 `get_template()` 方法读取
- 🔴 所有调用 `PromptService.get_template()` 的服务层代码都无法享受自定义模板功能
- 🟡 当前不影响基本功能（系统默认模板仍可正常工作）
